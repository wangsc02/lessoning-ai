# 从零到本地跑通：后端开发环境全配置

> **TL;DR**：这篇分两部分。前半部分讲公司项目为什么长这样（conan-parent、多模块结构），后半部分是一个完整的"第一天实操手册"，从装环境到项目跑起来，每一步可以直接复制命令执行。

## 公司项目的工程背景

在进入实操之前，先理解三件事。这些不是"背景知识"，而是直接决定你后面每一步操作为什么要那样做。

### 架构的核心目标：管理复杂度

公司服务端架构设计的第一要务不是"用什么技术栈"，而是**管理系统复杂度**。为此确立了五条优先级原则：

1. **服务接口的合理与简单** 高于 具体功能实现的优雅
2. **易于迭代** 高于 强大的预先设计
3. **团队和系统的自治** 高于 功能复用
4. **风格的简单一致** 高于 追求技术完美
5. **工具和自动化** 高于 灵活与精确

这些原则直接解释了后面你会看到的很多"限制"：为什么不能自己指定依赖版本、为什么项目结构长得都一样、为什么用 `fct` 而不是自己写脚本。它们不是形式主义，而是在用工具和约定把复杂度收住。

### conan-parent：把工程约束前置到项目起点

所有服务端项目必须使用 `conan-parent` 作为父项目，不允许自行指定基础组件版本。

```xml
<parent>
    <groupId>com.fenbi.conan</groupId>
    <artifactId>conan-parent</artifactId>
    <version>xxx</version>
</parent>
```

`conan-parent` 的作用不只是"帮你管版本"，它本质上是把工程约束前置到了项目起点：

- 依赖版本、RPC 框架、日志组件、鉴权 SDK 都已经被统一收住
- v0.1.91 之后保持完全向前兼容，可以直接改版本号升级
- 项目起不来时，不要第一反应手改依赖版本——大概率会越改越乱

详见：[conan-parent 介绍与使用](https://confluence.zhenguanyu.com/pages/viewpage.action?pageId=173035925)

### 项目结构：三层架构设计

公司采用 Monorepo 多模块架构，一个项目从宏观到微观分三层：

| 层级 | 语义边界 | 对应代码结构 |
|---|---|---|
| **项目（Project）** | 完整系统，一个团队独立负责，3-7 个服务端研发 | 一个 Git 仓库 |
| **服务/模块（Module）** | 运行时单位，伸缩性和 SLA 边界 | Maven Module |
| **组件（Component）** | 业务功能单元，强一致性边界 | backend 下的 Java Package |

一个项目里的模块通常按这样划分：

| 模块 | 职责 | 能否独立启动 |
|---|---|---|
| `{project}-web` | C 端 HTTP 入口 | 能，通常从这里启动 |
| `{project}-admin` | B 端 HTTP 入口 | 能 |
| `{project}-rpc` | RPC 服务入口 | 能 |
| `{project}-consumer` | 消息消费 | 能 |
| `{project}-job` | 定时任务 | 能 |
| `{project}-backend` | 核心业务逻辑 | 不能，是 library |
| `{project}-common` | 公共定义（thrift、枚举等） | 不能，是 library |
| `{project}-client` | RPC 客户端，供其他服务依赖 | 不能，是 library |

典型目录结构：

```
conan-mentor-xxx/
├── pom.xml                          # 父 pom
├── conan-mentor-xxx-common/         # 公共模块（thrift 定义等）
├── conan-mentor-xxx-client/         # RPC 客户端
├── conan-mentor-xxx-backend/        # 核心业务逻辑
│   └── component/                   # 业务组件（每个组件是一个业务单元）
├── conan-mentor-xxx-web/            # C端 HTTP 入口 ← 通常从这里启动
├── conan-mentor-xxx-admin/          # B端 HTTP 入口
├── conan-mentor-xxx-consumer/       # 消息消费
└── conan-mentor-xxx-job/            # 定时任务
```

几条关键规则：

- **启动时要进入 service module**（如 `xxx-web`），不要在项目根目录启动
- **模块间不允许循环依赖**
- **组件内通过数据库事务保证强一致性，组件间保证最终一致性**
- **数据表只允许一个组件访问**，不能跨组件直接读写

详见：[斑马服务架构设计指引](https://confluence.zhenguanyu.com/pages/viewpage.action?pageId=126295105)

## 第一天实操手册

以下每一步都可以直接复制命令执行。按顺序走完，项目应该能在本地跑起来。

### 1. 安装 IDEA

IntelliJ IDEA 是后端项目的默认 IDE，所有后续操作（装 JDK、导入项目、启动调试）都在这里完成。

建议直接在淘宝买一个正版账号，成本很低（通常几十块一年），省去折腾破解版的时间和风险。

下载地址：https://www.jetbrains.com/idea/download/

**必装插件**（安装完 IDEA 后，`Preferences` -> `Plugins` 搜索安装）：

| 插件 | 用途 | 优先级 |
|---|---|---|
| **Lombok** | 支持 Lombok 注解，不装会满屏报错 | 必装 |
| **Thrift** | `.thrift` 文件语法高亮和跳转 | 必装 |
| **Maven Helper** | 依赖冲突分析，排查版本问题时非常有用 | 必装 |
| **GitLab** | 在 IDEA 里直接做 Code Review、查看 MR | 必装 |
| **Spring Boot** | Spring Boot 配置补全、Bean 导航 | 推荐 |
| **Translation** | 选中代码或变量名一键翻译，读不熟悉的代码时很有帮助 | 推荐 |
| **PlantUML** | 在 IDEA 里预览 `.puml` 架构图和时序图 | 推荐 |
| **Nginx** | Nginx 配置文件语法高亮 | 可选 |
| **GitHub Copilot** | AI 代码补全 | 可选 |

### 2. 在 IDEA 里安装 JDK 8 + JDK 17

公司项目标准使用 **JDK 8**，部分新项目可能需要 JDK 17。建议两个都装。

直接在 IDEA 里下载最省事：

1. 打开 IDEA -> `File` -> `Project Structure` -> `SDKs` -> 点击 `+` -> `Download JDK`
2. 选择版本 `1.8`（JDK 8）和 `17`（JDK 17），Vendor 选 `Eclipse Temurin` 或 `Oracle OpenJDK`
3. IDEA 会自动下载到 `~/Library/Java/JavaVirtualMachines/` 下

下载完成后，在 `~/.zshrc` 中添加切换函数，方便命令行场景下切换版本（路径改成你实际的安装路径）：

```bash
# 默认使用 JDK 8
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_271.jdk/Contents/Home

# JDK 版本快速切换
jdk8() {
    export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_271.jdk/Contents/Home
    echo "已切换到 JDK 8"
    java -version
}

jdk17() {
    export JAVA_HOME=$HOME/Library/Java/JavaVirtualMachines/temurin-17.0.17/Contents/Home
    echo "已切换到 JDK 17"
    java -version
}

jdkv() {
    echo "当前 JAVA_HOME: $JAVA_HOME"
    java -version
}
```

```bash
source ~/.zshrc

# 验证
jdkv
jdk8    # 切到 8
jdk17   # 切到 17
jdk8    # 切回 8（日常默认）
```

> 路径要改成你实际的 JDK 安装路径，可以用 `ls ~/Library/Java/JavaVirtualMachines/` 查看。

### 3. 安装 Maven + Thrift

Maven 是包管理工具（类似 npm），Thrift 是 RPC 接口编译工具（公司只兼容 0.9.3）。

辅导服务团队（@shengtianyu、@zhengxiaohong、@jiayongpu）已经为前端转后端的同学准备了一套完整的环境配置文档和自动化脚本，放在 `conan-mentor-base` 项目里。先把这个项目 clone 下来，后面很多工具和文档都会用到：

```bash
git clone git@gitlab-ee.zhenguanyu.com:shengtianyu/conan-mentor-base.git ~/Conan/conan-mentor-base
```

然后用项目里提供的一键安装脚本（JDK 已经在 IDEA 里装好了，脚本只负责 Maven 和 Thrift）：

```bash
cd ~/Conan/conan-mentor-base/doc/java-backend-guide-for-frontend/dev-env-setup/tools
chmod +x setup-java-env.sh
./setup-java-env.sh
```

脚本会自动完成以下操作：

1. 检查 Homebrew 是否已安装
2. 用 Homebrew 安装 Maven，并把公司 `settings.xml` 复制到 `~/.m2/`
3. 交互式输入 LDAP 账号密码，自动写入 `settings.xml`
4. 配置 `MAVEN_OPTS` 环境变量
5. 验证 Maven 安装和仓库连接
6. 添加公司内部 Thrift tap，安装 Thrift 0.9.3
7. 配置 Thrift 到 PATH 环境变量
8. 验证 Thrift 版本

脚本运行结束后会显示验证结果。如果看到 `✓ Maven 安装和配置成功！` 和 `✓ Thrift 安装和配置成功！`，说明安装完成。

**如果脚本失败，手动安装**：

```bash
# Maven
brew install maven
cp ~/Conan/conan-mentor-base/doc/java-backend-guide-for-frontend/dev-env-setup/tools/settings.xml ~/.m2/
vim ~/.m2/settings.xml   # 修改 <username> 和 <password> 为 LDAP 账号密码
echo 'export MAVEN_OPTS="-Xms256m -Xmx1024m -Dfile.encoding=UTF-8"' >> ~/.zshrc

# Thrift
brew tap kanyun/thrift https://gitlab-ee.zhenguanyu.com/infra/microservice/homebrew-thrift
brew install kanyun/thrift/thrift@0.9
echo 'export PATH="/opt/homebrew/opt/thrift@0.9/bin:$PATH"' >> ~/.zshrc

source ~/.zshrc

# 验证
mvn -version
thrift -version   # 应输出 Thrift version 0.9.3
```

### 4. 安装 conan-knowledge（斑马知识库）

`conan-knowledge` 是斑马团队维护的集中式知识库，安装后所有项目自动获得统一的 commands 和 rules。

它能提供的能力包括：

| 命令 | 用途 |
|---|---|
| `/create-feature-branch` | 创建功能分支并自动创建 MR |
| `/commit-push-mr` | 自动生成 commit message 并推送创建 MR |
| `/sonar-fix` | 自动修复 SonarQube 代码质量问题 |
| `/analyze-alarm` | 根据 alarmId/traceId 智能分析告警和日志 |
| `/analyze-logs` | 根据 traceId 分析报错日志并提供修复建议 |
| `/console-build` | 快速部署项目到测试环境 |

同时还会自动加载 Java、TypeScript、React 等编码规范 rules，让 Cursor 在写代码时遵循团队约定。

```bash
# clone 项目
git clone git@gitlab-ee.zhenguanyu.com:shengtianyu/conan-knowledge.git ~/Conan/conan-knowledge

# 安装（推荐全局模式，一次安装所有项目生效）
cd ~/Conan/conan-knowledge
./install-global.sh
```

安装后无需额外配置，切换分支时会自动在后台更新知识库。

详见：`~/Conan/conan-knowledge/README.md` 和 `~/Conan/conan-knowledge/QUICK_START.md`

### 5. 安装 fct 并拉项目

`fct`（fenbi-code-tool）是公司内部命令行工具，用于拉项目、推代码、编译 thrift、管理分支。

```bash
# 安装
git clone ssh://gerrit.zhenguanyu.com:29418/fenbi-code-tool
cd fenbi-code-tool

# 获取 refresh token（打开下面的链接，复制 token）
# https://console.zhenguanyu.com/#/management/projects/infra-garage/settings/refresh-tokens

sh bin/fct init <你的ldap账号> <refresh_token>

# 验证
fct init --validate
```

```bash
# 拉项目
fct clone <project-name>
```

### 6. 导入项目到 IDEA

1. `File` -> `Open` -> 选择项目根目录（包含 `pom.xml`）
2. 等待 Maven 下载依赖（右下角进度条）
3. 确认 JDK：`File` -> `Project Structure` -> `Project SDK` 选择 JDK 8

### 7. 本地启动项目

Spring Boot 项目在 IDEA 里启动非常简单：

1. 在 IDEA 左侧项目树中，找到要启动的 service module（如 `xxx-web`）
2. 找到启动类 `*Application.java`（带 `@SpringBootApplication` 注解）
3. IDEA 会自动识别它是可运行的 main class，直接点击类名或 `main` 方法旁边的绿色三角，选择 `Run` 或 `Debug` 即可

日常开发基本就这一步，不需要手动执行任何命令。

**需要调整时**：

- 指定 profile：修改项目 `application.yaml` 中的 `spring.profiles.active`，或在 `Run Configuration` -> `Active profiles` 中填写
- 端口冲突：在 `Run Configuration` -> `VM Options` 中加 `-Dserver.port=8081`
- 多个 service module 要同时启动：每个 module 的启动类分别点击运行即可，IDEA 支持同时运行多个

### 8. 验证项目跑起来了

日志刷起来不等于启动成功。看到以下日志内容才算真正跑起来了：

```
Started XxxApplication in x.xxx seconds (JVM running for x.xxx)
```

或者包含类似内容：

```
Tomcat started on port(s): 8080 (http)
```

如果日志中出现异常堆栈（`Exception`、`Error`）并且进程退出了，说明启动失败。

启动成功后，再用 curl 验证最小请求：

```bash
curl http://localhost:8080/<project-name>/api/health
```

确认能返回正常响应（HTTP 200），而不是 404 或连接拒绝。如果项目依赖数据库或 Redis，也检查一下启动日志中有没有连接失败的警告。

**启动失败时按这个顺序排查**：

1. 父工程是不是 `conan-parent`，版本对不对
2. 有没有手动指定了不该指定的依赖版本
3. JDK 版本是不是 8（用 `jdkv` 确认）
4. Maven `settings.xml` 里 LDAP 账号密码对不对
5. 是不是在项目根目录启动的（应该进入 service module）

### 9. fct 常用命令速查

```bash
# 创建功能分支
fct remote-branch -b feature/<author>-<date>-<description> -L

# 推代码（会自动跑 lint 检查）
fct push -S

# 编译 thrift 文件
fct ct --private-field --proxy

# 查看项目版本
fct version

# 升级版本号
fct uv 1.0.1-SNAPSHOT
```

## 常见问题

### Maven 下载依赖失败

```bash
mvn help:effective-settings   # 检查配置是否生效
```

大概率是 `~/.m2/settings.xml` 中 LDAP 账号密码不对。

### Thrift 版本不是 0.9.3

```bash
brew install kanyun/thrift/thrift@0.9
echo 'export PATH="/opt/homebrew/opt/thrift@0.9/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
thrift -version
```

### IDEA 满屏红色报错

大概率是 Lombok 插件没装，或 Maven 依赖没下载完。先装 Lombok 插件，再在 IDEA 中 `Maven` -> `Reload All Maven Projects`。

### 启动时端口被占用

```bash
lsof -i :8080   # 查看占用进程

# 换端口启动
# IDEA: VM Options 加 -Dserver.port=8081
# 命令行: fct run <profile> -Dserver.port=8081
```
