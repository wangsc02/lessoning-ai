# 结构化头脑风暴

> 用结构化方法分析问题，避免遗漏关键因素，得出可执行的结论

## 📋 使用场景

- 技术选型（选框架、选工具、选架构）
- 方案设计（如何实现某个功能）
- 问题诊断（系统故障、性能问题）
- 决策分析（做还是不做，现在做还是以后做）

## 🎯 分析框架（6 个维度）

我会引导你按以下 6 个维度系统思考：

### 1. 问题定义（What & Why）
- **核心问题是什么？** 用一句话描述
- **为什么要解决它？** 不解决会怎样？
- **成功的标准是什么？** 如何判断问题已解决？

### 2. 现状分析（As-Is）
- **当前是什么情况？** 现有方案/系统的状态
- **痛点在哪里？** 具体的不足或限制
- **已有哪些尝试？** 之前做过什么努力

### 3. 方案对比（Options）
- **方案 A / B / C** 是什么？（至少列 2-3 个）
- **每个方案的核心思路**
- **技术栈 / 实现路径**

### 4. 多维度评估（Evaluation）

为每个方案评估以下维度（打分 1-5 星）：

| 维度 | 方案 A | 方案 B | 方案 C | 权重 |
|------|--------|--------|--------|------|
| 技术可行性 | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ | 高 |
| 实施成本（时间） | ⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ | 高 |
| 长期可维护性 | ⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 中 |
| 团队熟悉度 | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐ | 中 |
| 扩展性 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐ | 中 |
| 风险程度（越低越好） | ⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ | 高 |

### 5. 风险与限制（Risks & Constraints）
- **每个方案的主要风险是什么？**
- **有哪些硬性限制？**（时间、预算、技术、人员）
- **如何降低风险？**（应对策略）

### 6. 决策建议（Decision）
- **推荐方案**：哪个？为什么？
- **实施路径**：分几步？每步做什么？
- **验收标准**：如何验证是否成功？
- **回退方案**：如果失败，Plan B 是什么？

## 💡 使用技巧

### 技巧 1：从小问题开始
不要一上来就想"完美方案"，先列出所有可能的选项，再逐步筛选。

### 技巧 2：量化评估
尽量用数字（时间、成本）而非"很快"、"很贵"这种模糊表述。

### 技巧 3：写下假设
明确你的假设（如"假设团队有 2 周时间"），后续评估才有意义。

### 技巧 4：反向思考
除了问"为什么选 A"，也要问"为什么不选 B"。

## 🎯 典型应用示例

### 示例 1：技术选型
```
问题：选择 Web 应用的前端框架（React vs Vue vs Svelte）
方案：React / Vue / Svelte
维度：生态、学习曲线、性能、团队熟悉度、招聘难度
```

### 示例 2：架构决策
```
问题：如何实现实时通信（WebSocket vs SSE vs 长轮询）
方案：三种协议的对比
维度：浏览器兼容性、服务器负载、实现复杂度、成本
```

### 示例 3：流程优化
```
问题：如何提升 CI/CD 速度（当前 30 分钟太慢）
方案：并行构建 / 缓存优化 / 增量构建
维度：提升幅度、实施成本、稳定性、副作用
```
