```mermaid
graph LR
    A[收集材料] --> B[/generate-learning-doc]
    B --> C[检查和修改]
```

```mermaid
graph LR
```

4. **可选**：用 `/generate-learning-doc` 将决策过程文档化

## 📚 命令详细说明

### `/brainstorm` - 结构化头脑风暴

**何时使用**：
- ✅ 技术选型（选框架、选工具、选架构）
- ✅ 方案设计（如何实现某个功能）
- ✅ 问题诊断（系统故障、性能问题）
- ✅ 决策分析（做还是不做，现在做还是以后做）

**输出**：
- 6 维度分析报告
- 方案对比表格
- 决策建议

**实施成本**：零（纯提示词模板）

---

### `/generate-learning-doc` - 生成学习文档

**何时使用**：
- ✅ 学习新技术后想整理笔记
- ✅ 技术调研需要形成文档
- ✅ 框架对比需要结构化输出

**输出**：
- 完整的 Markdown 学习文档
- 包含：TL;DR、核心概念、对比、案例、决策框架

**特点**：
- 一键生成终稿（不需要大纲确认）
- 自动质量检查
- 符合 `technical-writing-quality` 规范

---

### `/publish-knowledge` - 发布知识到 GitHub

**何时使用**：
- ✅ 文档编写完成，需要提交
- ✅ 文档中包含 Mermaid 图表需要渲染
- ✅ 想要自动生成 commit message

**功能**：
1. 检测 Mermaid 代码块
2. 生成高清流程图（2000px，3x scale）
3. 智能生成 commit message
4. 提交并推送到 GitHub
5. 验证推送成功

**前置条件**：
- Git 仓库状态正常
- 有修改的 `knowledge/**/*.md` 文件

## 🌟 使用技巧

### 技巧 1：善用 Brainstorm

在写代码前，先用 `/brainstorm` 理清思路，可以避免：
- 方案选错（后期推倒重来）
- 遗漏风险（上线后踩坑）
- 团队争议（没有系统分析依据）

### 技巧 2：文档化决策

用 `/brainstorm` 分析完后，用 `/generate-learning-doc` 将决策过程文档化：
- 方便后续复盘
- 团队新成员能快速了解"为什么这样设计"
- 作为技术债务的记录

### 技巧 3：Mermaid 自动渲染

在文档中使用 Mermaid 画图，`/publish-knowledge` 会自动：
- 生成高清 PNG
- 替换为图片链接
- 保证 GitHub 上正确显示

## 🆘 常见问题

### Q: 为什么要用 `/brainstorm` 而不是直接问 Claude？

**A**: 区别在于结构化
- 直接问：容易遗漏维度，答案可能不完整
- 用 Brainstorm：强制 6 维度分析，输出标准化，可对比

### Q: `/generate-learning-doc` 和手写文档的区别？

**A**: 速度和质量
- 手写：可能忘记某些部分（如边界条件、反模式）
- 用命令：自动包含所有必要章节，符合质量规范

### Q: 我能改命令吗？

**A**: 当然！
- 所有命令都在 `.cursor/commands/` 目录
- 直接编辑 `.md` 文件即可
- 重启 Cursor 后生效
