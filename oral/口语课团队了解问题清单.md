# 口语课团队了解问题清单

> 接手新团队前的深度调研问题，用于一对一沟通、团队会议、跨团队协作时使用

---

## 一、团队与人员

| 问题 | 目的 |
|------|------|
| 团队目前各端（服务端/前端/客户端）分别有多少人？各自负责哪些模块？ | 了解人力分布和职责边界 |
| 团队内谁是技术骨干？谁对业务最熟悉？ | 识别关键人物，建立信任 |
| 之前客户端 leader 离开后，团队氛围有没有变化？有没有人有离开的想法？ | 评估稳定性风险 |
| 团队成员对"全栈化"的态度如何？有没有人尝试过跨端开发？ | 评估推进全栈的阻力 |
| 目前谁在带 Scrum？Sprint 周期是多长？ | 了解现有管理节奏 |

## 提问策略

### 原则
1. **先问人，再问事**：团队稳定性是第一优先级，技术可以学，人走了就真没了
2. **先问现状，再问痛点**：避免一上来就"指点江山"，先听懂再说
3. **通过问题建立信任**：好的问题本身就是在展示专业度
4. **把问题变成对话**：不要像审问，而是"我想了解一下..."，让对方感受到被尊重

### 分场景使用

| 场景 | 重点问题 |
|------|----------|
| 与核心骨干 1:1 | 团队氛围、技术痛点、个人发展期望 |
| 与产品同学沟通 | 需求流程、历史协作问题、对研发的期望 |
| 与 AI 团队对接 | 接口边界、Prompt 工程、能力内化意愿 |
| 与教研团队了解 | 内容规范、对技术的要求、协作频率 |
| 团队全员会议 | 整体业务目标、近期规划、大家的困惑 |
